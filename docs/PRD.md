## 1. 产品概述

### 1.1 产品背景

哥伦比亚大学研究生群体中，相当一部分住在 **Fort Lee, NJ**。每日通勤需要经过 **乔治·华盛顿大桥（George Washington Bridge, GWB）**，单程过桥费 **$23.30**。

**问题**：
- 独自开车：每天 $46.6 过桥费，成本高昂
- 公交出行：耗时 1.5-2 小时，不灵活
- 微信群拼车：效率低，信任难建立

**机会**：
- 学生群体有强烈的拼车需求
- 固定路线（Fort Lee ↔ Columbia）适合拼车
- 通过分摊过桥费可以实现双赢

### 1.2 产品定义

面向哥伦比亚大学留学生的**微信小程序**，提供 Fort Lee ↔ Columbia 通勤拼车服务。

**一句话描述**：让哥大学生轻松拼车，省钱又省时。

### 1.3 目标用户

| 用户画像 | 特征 |
|---------|------|
| 主力用户 | 住在 Fort Lee 的哥大研究生 |
| 出行场景 | 每日通勤、往返校园 |
| 核心需求 | 省钱、准时、信任 |
| 支付意愿 | 愿为拼车支付 $8-15 |

### 1.4 成功指标

| 指标 | 目标值 |
|------|--------|
| 周活跃用户 | 50+ |
| 单日拼车成功订单 | 20+ |
| 平均评分 | 4.5+ |
| 订单完成率 | 90%+ |
| 用户留存率（周） | 60%+ |

## 2. 功能需求

### 2.1 核心功能

#### 2.1.1 首页（拼车广场）

**功能描述**：展示所有可加入的拼车行程

**页面元素**：

| 元素 | 说明 |
|------|------|
| 地图区域 | 显示 Fort Lee ↔ Columbia 路线 |
| 今日统计 | 「今天已有 X 人同行」 |
| 模式切换 | 顺风车 / Uber 拼单 |
| 行程列表 | 可加入的拼车行程卡片 |
| 发布按钮 | FAB 悬浮按钮，点击发布行程 |

**交互逻辑**：

```
用户进入首页
    ↓
加载地图和行程列表
    ↓
用户可切换模式（顺风车/Uber）
    ↓
点击行程卡片 → 进入行程详情
点击 FAB → 进入发起拼车
下拉刷新 → 重新加载列表
```

#### 2.1.2 行程详情

**功能描述**：展示单个行程的完整信息

**页面元素**：

| 元素 | 说明 |
|------|------|
| 车主信息 | 头像、姓名、认证标识、评分、行程次数、车型车牌 |
| 路线时间轴 | Fort Lee → GWB → Columbia，可视化展示 |
| 费用明细 | GWB $8，含过桥费说明 |
| 座位状态 | 已满/剩余座位、乘客列表 |
| 加入按钮 | 底部固定按钮 |

**交互逻辑**：

```
用户点击行程卡片
    ↓
跳转行程详情页
    ↓
加载车主信息、路线、费用、乘客
    ↓
用户阅读详情
    ↓
点击「加入拼车」→ 跳转确认页
点击返回 → 返回首页
```

#### 2.1.3 发起拼车

**功能描述**：车主发布自己的行程

**页面元素**：

| 元素 | 说明 |
|------|------|
| 模式切换 | 顺风车 / Uber 拼单 |
| 出发地/目的地 | 下拉选择，Fort Lee / Columbia |
| 切换按钮 | 一键互换出发地和目的地 |
| 出发时间 | 时间选择器 |
| 人数选择 | 1-4 人可选 |
| 费用说明 | 固定 $8/人，自动计算 |
| AI 建议 | 智能推荐出发时间 |

**交互逻辑**：

```
用户点击 FAB 发布按钮
    ↓
弹出发起拼车 Bottom Sheet
    ↓
用户选择模式
    ↓
用户设置路线和时间
    ↓
（可选）点击 AI 建议，获取推荐时间
    ↓
点击「发起拼车」
    ↓
创建行程 → 跳转我的行程
```

#### 2.1.4 我的行程

**功能描述**：管理用户已加入或发布的行程

**页面元素**：

| 元素 | 说明 |
|------|------|
| Tab 切换 | 进行中 / 历史 |
| 行程卡片 | 时间、路线、车主/乘客、状态 |
| 状态标签 | 待出发 / 进行中 / 已完成 / 已取消 |

**交互逻辑**：

```
用户点击底部「我的行程」
    ↓
显示进行中 Tab
    ↓
切换 Tab 查看历史
    ↓
点击行程卡片 → 查看详情
点击取消 → 取消行程（车主权限）
```

#### 2.1.5 确认加入

**功能描述**：确认并支付拼车费用

**页面元素**：

| 元素 | 说明 |
|------|------|
| 车主信息 | 姓名、认证、评分 |
| 行程概要 | 时间、路线、日期 |
| 乘客列表 | 已加入的乘客 |
| 费用明细 | GWB $8 |
| 支付方式 | Zelle / Venmo |
| 确认按钮 | 确认并支付 |

**交互逻辑**：

```
用户点击「加入拼车」
    ↓
跳转确认页
    ↓
用户选择支付方式
    ↓
点击「确认加入」
    ↓
跳转支付（外部 App）
    ↓
返回 → 加入成功
```

### 2.2 辅助功能

#### 2.2.1 用户认证

**功能描述**：验证用户为哥伦比亚大学学生

**流程**：

```
用户注册
    ↓
进入认证流程
    ↓
输入哥伦比亚大学邮箱
    ↓
接收验证码
    ↓
验证通过
    ↓
认证标识显示
```

**认证标识**：
- 哥大认证徽章（蓝色）
- 显示「Columbia Verified」

#### 2.2.2 AI 出发时间建议

**功能描述**：根据历史数据推荐最佳出发时间

**数据来源**：
- 历史行程的出发时间
- GWB 交通规律
- 天气因素（未来）

**推荐策略**（v1.0 heuristic_v1）：

```javascript
{
  // 早高峰策略
  if (departureHour >= 7 && departureHour <= 9) {
    confidence: "中",
    reason: "早高峰时段，建议提前出发"
  }
  // 最佳时间
  if (departureHour == 8 && departureMinute == 30) {
    confidence: "高",
    reason: "历史数据显示该时间出发准时率最高"
  }
}
```

## 3. 数据需求

### 3.1 数据模型

#### 3.1.1 用户（users）

```javascript
{
  _id: "user_xxx",
  name: "张三",
  avatar: "https://...",
  phone: "1234567890",
  cuEmail: "zhangsan@columbia.edu",
  isVerified: true,
  rating: 4.9,
  tripCount: 86,
  createdAt: "2026-01-01",
  updatedAt: "2026-04-01"
}
```

#### 3.1.2 行程（trips）

```javascript
{
  _id: "trip_xxx",
  driverId: "user_xxx",
  mode: "rideshare", // "rideshare" | "uber"
  route: {
    from: "fort_lee",
    to: "columbia"
  },
  departureTime: "2026-04-01T08:30:00Z",
  availableSeats: 3,
  totalSeats: 4,
  pricePerPerson: 8,
  status: "active", // "active" | "completed" | "cancelled"
  createdAt: "2026-04-01T00:00:00Z"
}
```

#### 3.1.3 乘客（passengers）

```javascript
{
  _id: "passenger_xxx",
  tripId: "trip_xxx",
  passengerId: "user_xxx",
  status: "confirmed", // "confirmed" | "left"
  paymentMethod: "zelle",
  createdAt: "2026-04-01T00:00:00Z"
}
```

### 3.2 数据统计

| 指标 | 计算方式 |
|------|---------|
| 今日拼车人数 | COUNT(trips WHERE date = today AND status = completed) |
| 用户评分 | AVG(ratings WHERE userId = driverId) |
| 行程完成率 | COUNT(completed) / COUNT(total) |
